yuzhou-mac-theme: Terminal Setup Guide

Author

Yuzhou Chang

Published

May 23, 2026

Overview

This guide walks through setting up a clean, fast, cross-platform terminal on macOS and Linux using:

  • Ghostty — modern GPU-accelerated terminal
  • zsh with autosuggestions + syntax highlighting
  • Starship — fast cross-shell prompt
  • eza — ls replacement with colors and icons
  • JetBrainsMono Nerd Font — monospaced font with developer icons
Note

All code, configs, and the one-shot installer live at github.com/BMEngineeR/yuzhou-mac-theme.

One-shot install

git clone https://github.com/BMEngineeR/yuzhou-mac-theme.git ~/yuzhou-mac-theme
cd ~/yuzhou-mac-theme
bash install.sh

The installer is idempotent — re-running it is safe. Existing ~/.zshrc and ~/.config/ghostty/config are backed up with a timestamp before being replaced by symlinks into the repo.

What you get

Starship prompt

The Starship prompt automatically shows:

  • Current directory (with shortened paths)
  • Git branch and status
  • Active language version (when in a project: Node, Python, Rust, Go, …)
  • Command execution time when commands take longer than a threshold
  • Non-zero exit status from the previous command

Example interactive session:

~ on  main
❯ cd Documents/GitHub/yuzhou-mac-theme

yuzhou-mac-theme on  main [!?]
❯ git status --short
 M README.md
?? docs/

yuzhou-mac-theme on  main [!?] took 3ms
❯

The leading icons (, ``) render as proper glyphs in any Nerd-Font-enabled terminal.

Colored ls with eza

The aliases below replace ls:

alias ls='eza --icons --group-directories-first'
alias ll='eza -lh --icons --group-directories-first --git'
alias la='eza -lah --icons --group-directories-first --git'
alias lt='eza --tree --level=2 --icons --group-directories-first'

Color coding by file type:

Type Color Examples
Directory Blue src/, node_modules/
Executable script Green install.sh, *.py with +x
Symlink Cyan current -> v1.2.3/
Archive Red *.zip, *.tar.gz
Image Magenta *.png, *.jpg
Plain file White README.md, *.txt

Syntax highlighting & autosuggestions

As you type, zsh-syntax-highlighting colors valid commands green and invalid ones red, while zsh-autosuggestions shows a grayed-out suggestion from your history that you accept by pressing →.

Example: after running git status --short once, typing git st shows the rest of the previous command in dim gray, ready to accept.

Customization

Switching the color theme

List built-in themes:

ghostty +list-themes

Edit ~/.config/ghostty/config:

theme = "Catppuccin Mocha"

Popular options:

  • Catppuccin Mocha
  • Tokyo Night
  • Dracula
  • Nord
  • Solarized Dark
  • GruvboxDark (default)

Customizing Starship

Generate a starter config:

mkdir -p ~/.config
starship preset nerd-font-symbols -o ~/.config/starship.toml

Then edit ~/.config/starship.toml. The Starship docs list every module: https://starship.rs/config/.

Customizing eza colors

Set EZA_COLORS in ~/.zshrc to override defaults. Example — make directories bold magenta:

export EZA_COLORS="di=1;35"

Full syntax: https://github.com/eza-community/eza/blob/main/man/eza_colors.5.md.

Verification

After running the installer and exec zsh, confirm everything is wired up:

# Prompt — Starship should be active
echo $STARSHIP_SHELL          # "zsh"

# eza — should print a version
eza --version | head -1

# Plugins — these are populated when the plugins are loaded
typeset -p ZSH_HIGHLIGHT_VERSION 2>&1 | head -1
typeset -p ZSH_AUTOSUGGEST_STRATEGY 2>&1 | head -1

# Font — Ghostty needs Nerd Font for icons to render
fc-list 2>/dev/null | grep -i "JetBrainsMono Nerd" | head -1

Troubleshooting

WarningIcons render as squares or “?”

The terminal is not using a Nerd Font. Fix:

  1. Confirm the font is installed: fc-list | grep -i nerd (Linux) or ls ~/Library/Fonts | grep -i nerd (macOS).
  2. Edit ~/.config/ghostty/config and set font-family = "JetBrainsMono Nerd Font".
  3. Restart Ghostty (font changes do not hot-reload reliably).
Warningcommand not found: starship

Either Starship didn’t install, or your PATH doesn’t include its install location. Check:

# macOS / Linuxbrew
ls $(brew --prefix)/bin/starship

# Linux native install (from starship.rs installer)
ls ~/.local/bin/starship /usr/local/bin/starship

Re-run the installer; it is idempotent.

WarningPlugins not loading

Run zsh -xv 2>&1 | grep -i 'no such file' | head to see which path the source call tried. Then check the appropriate location for your package manager (see the comments in zshrc).
